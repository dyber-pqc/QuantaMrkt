# research-assistant

![PQC Native](https://img.shields.io/badge/PQC-Native-7c3aed) ![ML-DSA-65](https://img.shields.io/badge/ML--DSA-65-0ea5e9) ![Apache 2.0](https://img.shields.io/badge/license-Apache%202.0-green)

## What it does

`research-assistant` is a RAG agent with a trust boundary: it only
reads chunks that carry a valid ML-DSA signature from the corpus
ingest pipeline, and it re-signs every summary it produces as a
C2PA-compatible `ContentManifest`. This prevents poisoning attacks
against the retrieval layer and gives downstream readers a cryptographic
chain from corpus chunk to final summary.

## Tools it composes

- [`pqc-rag-signing`](/tools/pqc-rag-signing) - PQC-signed chunks for vector stores
- [`pqc-content-provenance`](/tools/pqc-content-provenance) - C2PA manifests with ML-DSA signatures

## Quick start

```bash
pip install pqc-agent-research-assistant
python agent.py
```

## Identity

The DID from `identity.json` is a placeholder. The real ML-DSA-65
keypair is generated by the `seed-agents.yml` GitHub Action using
`liboqs`, then platform-signed with the `did:web:quantamrkt.com`
ML-DSA-87 root key before being written into this file.
