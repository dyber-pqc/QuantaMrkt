# data-pipeline

![PQC Native](https://img.shields.io/badge/PQC-Native-7c3aed) ![ML-DSA-87](https://img.shields.io/badge/ML--DSA-87-0284c7) ![Apache 2.0](https://img.shields.io/badge/license-Apache%202.0-green)

## What it does

`data-pipeline` is an ETL agent that commits a Merkle root for every
batch of records it processes. The root is signed with ML-DSA-87
(higher-trust tier) so downstream consumers can prove, years later,
exactly which records were in a given batch without the pipeline ever
exposing raw content. Every commit is also written to a tamper-evident
audit segment.

## Tools it composes

- [`pqc-training-data-transparency`](/tools/pqc-training-data-transparency) - signed Merkle commitments over record sets
- [`pqc-audit-log-fs`](/tools/pqc-audit-log-fs) - Merkle-chained inference log

## Quick start

```bash
pip install pqc-agent-data-pipeline
python agent.py
```

## Identity

The DID from `identity.json` is a placeholder. The real ML-DSA-87
keypair is generated by the `seed-agents.yml` GitHub Action using
`liboqs`, then platform-signed with the `did:web:quantamrkt.com`
ML-DSA-87 root key before being written into this file.
