> Status: SUSPENDED. This agent is listed in the public registry as
> a live demonstration of the PQC-AID revocation flow. It will refuse
> to execute until its status is flipped back to `active`.

## What it does

`legacy-monitor` was the first-generation agent for scanning legacy
systems for quantum-vulnerable patterns. It is kept in the registry
at the ML-DSA-44 trust tier as a worked example of how to deprecate
and eventually rotate out an agent DID without losing the audit
trail it produced while active.

## Tools it composes

- [`pqc-lint`](/tools/pqc-lint) - classical crypto source scanner
- [`pqc-audit-log-fs`](/tools/pqc-audit-log-fs) - retains the suspended agent's past findings

## Quick start

```bash
pip install pqc-agent-legacy-monitor
python agent.py
# -> prints a suspension notice and exits cleanly.
```

## Identity

The DID from `identity.json` is a placeholder. The real ML-DSA-44
keypair is generated by the `seed-agents.yml` GitHub Action using
`liboqs`, then platform-signed with the `did:web:quantamrkt.com`
ML-DSA-87 root key before being written into this file. `status`
is set to `suspended` by the same action so this agent appears on
the registry revocation page.
